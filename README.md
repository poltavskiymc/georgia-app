# Грузия — трэвел-компаньон

Оффлайн-PWA для поездки по Грузии на машине: ИИ-помощник, переводчик, разговорник,
гид по маршруту с навигатором, конвертер валют. Без сборщика — чистый HTML/CSS/JS,
раздаётся любым статическим сервером.

## Структура

```
georgia-app/
├── index.html            # разметка всех вкладок + подключение css/js по порядку
├── manifest.webmanifest  # PWA-манифест (иконки, имя, цвета)
├── sw.js                 # service worker: HTML — network-first, статика — cache-first
├── README.md             # этот файл
├── css/
│   └── styles.css        # все стили (переменные темы, светлая/тёмная)
├── js/                   # скрипты грузятся в этом порядке (глобальные функции, без модулей)
│   ├── data.js           # КОНТЕНТ: PHRASES, ROUTE, TIPS, SYSTEM. Только данные.
│   ├── util.js           # ХЕЛПЕРЫ: esc, byteLen/fmtBytes, fmtTs, haversine/fmtDist, speak
│   ├── traffic.js        # учёт трафика: traffic, addTraffic(), renderTraffic()
│   ├── nav.js            # переключение вкладок: show()
│   ├── phrases.js        # вкладка «Фразы» + избранное: renderPhrases(), favs
│   ├── route.js          # вкладка «Маршрут»: navLink(), applyDist()/OSRM, геолокация
│   ├── money.js          # вкладка «Деньги»: конвертер + fetchRate()
│   ├── chat.js           # вкладка «Чат»: send(), renderChat(), refreshKeyHint(), history
│   ├── translate.js      # вкладка «Перевод»: translate() (DeepSeek → en/ka + озвучка)
│   └── settings.js       # вкладка «Ключ»: ключ DeepSeek, сброс трафика, версия. Грузится ПОСЛЕДНИМ.
└── icons/                # иконки приложения
```

## Как это связано (важно при правках)

- **Порядок скриптов в `index.html` важен.** Модули общаются через глобальные функции,
  а не через import. Правило: файл можно двигать, пока функции, которые он *вызывает
  на загрузке*, определены в файлах выше. Цепочка зависимостей:
  `data → util → traffic → nav → phrases → route → money → chat → translate → settings`.
- Общие функции живут в **util.js** (`esc`, `speak`, `haversine`, `fmtTs`, `byteLen`…).
  Новую утилиту, нужную двум+ вкладкам, добавляй туда.
- Любой сетевой запрос оборачивай `addTraffic(byteLen(body), byteLen(raw))` — иначе
  вкладка «Ключ» покажет неполный расход.
- **Контент правится в `data.js`** (новые фразы, точки маршрута, советы) — трогать
  логику при этом не нужно.

## Кэш и версии

- При изменении **любого** файла в `css/` или `js/` — подними `CACHE` в `sw.js`
  (`georgia-vN` → `vN+1`) и метку `APP_VERSION` в `js/settings.js`, иначе телефон
  отдаст старую версию из кэша. Номер версии виден внизу вкладки «Ключ».
- `index.html` отдаётся network-first, поэтому подхватывается сразу (при наличии сети).

## Ограничения окружения

- **Геолокация** (расстояния на вкладке «Маршрут») и **service worker** (офлайн)
  в браузере работают только на **https или localhost**. По `http://<LAN-IP>` они
  блокируются. Для полноценной работы на телефоне — раздавать по https (напр. хостинг
  вроде Netlify/Cloudflare Pages) или через localhost.
- **Расстояния по дорогам** берутся с публичного демо-сервера OSRM
  (`router.project-osrm.org`) без ключа — годится для пары пользователей; при недоступности
  показывается расстояние по прямой (с «~»).
- **Распознавание по фото** пока не сделано: у DeepSeek API нет vision, нужна отдельная
  vision-модель (Gemini/OpenAI/Claude).

## Внешние сервисы

- **DeepSeek** (`api.deepseek.com`) — чат и переводчик. Ключ хранится в localStorage телефона.
- **open.er-api.com** — курс GEL→RUB.
- **router.project-osrm.org** — дорожные расстояния.
- Навигаторы: Google Maps, Яндекс.Карты, Maps.me (ссылки-маршруты, открываются во внешнем приложении).
