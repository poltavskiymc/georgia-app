# Грузия — трэвел-компаньон

Оффлайн-PWA для поездки по Грузии на машине: ИИ-помощник, переводчик, разговорник,
гид по местам и еде с навигатором, план поездки с чек-листами, конвертер валют.
Без сборщика — чистый HTML/CSS/JS, раздаётся любым статическим сервером.

## Вкладки

Пять вкладок внизу; настройки — ⚙️ в шапке (не вкладка).

| Вкладка | Что внутри |
|---|---|
| 💬 **Чат** | ИИ-помощник «Гоги» (DeepSeek), несколько чатов |
| 🗣️ **Язык** | под-вкладки: 🌐 Перевод и 🗣️ Фразы |
| 🧭 **Гид** | под-вкладки: 🗺️ Места (регионы, точки, навигатор, энциклопедия) и 🍽️ Еда (блюда) |
| ✅ **План** | 🧳 Сборы и граница · 📍 Хочу посмотреть · 🍽️ Хочу попробовать (+ оценка в хинкалях) |
| 💸 **Деньги** | конвертер лари ⇄ рубль, курс, практические советы |

«Гид» — источник, «План» — приёмник: у каждой точки и блюда есть кнопка ＋, которая
кладёт их в соответствующий список «Плана».

## Как это связано (важно при правках)

- **Порядок скриптов в `index.html` важен.** Модули общаются через глобальные функции,
  а не через import. Правило: файл можно двигать, пока функции, которые он *вызывает
  на загрузке*, определены в файлах выше. Цепочка зависимостей:
  `data → util → traffic → nav → phrases → plan → route → wiki → food → money → chat → translate → settings`.
- **`plan.js` грузится ДО `route.js` и `food.js`:** те при отрисовке спрашивают у него
  `planItem()`/`planRating()` и вешают `planToggle()`/`planRate()`. Обратная связь (пункт убрали
  или оценили прямо в «Плане») идёт событием `planchange` на `document` — гид перерисовывает
  свои кнопки. Состояние плана: `plan_see`, `plan_eat`, `pack_done` в localStorage;
  id пункта стабильный (координаты точки / название блюда), чтобы позже можно было
  мерджить план между двумя телефонами.
- Общие функции живут в **util.js** (`esc`, `speak`, `haversine`, `fmtTs`, `byteLen`…).
  Новую утилиту, нужную двум+ вкладкам, добавляй туда.
- Любой сетевой запрос оборачивай `addTraffic(byteLen(body), byteLen(raw))` — иначе
  вкладка «Ключ» покажет неполный расход.
- **Контент правится в `data.js`** (новые фразы, точки маршрута, советы) — трогать
  логику при этом не нужно.

## Кэш и версии

При изменении **любого** файла в `css/` или `js/` подними номер версии в **трёх** местах,
они должны совпадать:

1. `const VER = 'N'` в `sw.js` (из него собирается имя кеша `georgia-vN`);
2. `?v=N` у всех `<link>`/`<script>` в `index.html`;
3. `APP_VERSION` в `js/settings.js` (видно внизу ⚙️ — по этой строке проверяют, что телефон обновился).

Почему так (сборка 21 залипла у пользователя именно на этом):

- **HTML — network-first.** Есть сеть — всегда свежая разметка; нет сети — из кеша. Пока
  `index.html` отдавался из кеша, телефон мог неделю жить на старой версии и не узнать об обновлении.
- **css/js — stale-while-revalidate** (мгновенно из кеша, обновление в фоне), но подключаются
  с `?v=N`. Свежий HTML просит `js/plan.js?v=23` — такого URL в кеше нет, значит файл гарантированно
  приедет из сети. Залипнуть на старом js невозможно.
- Ручная кнопка «Обновить приложение» (⚙️) сносит все кеши и перезагружает — аварийный выход,
  если что-то всё же пошло не так.
- Офлайн полностью рабочий: без сети всё отдаётся из кеша, сетевые запросы (ИИ/курс/OSRM)
  просто не выполнятся до появления связи.

## Проверка правок без браузера

Headless Chrome в этом окружении зависает — не использовать. Вместо него: `node --check js/*.js`
плюс прогон настоящего `index.html` в **jsdom** (скрипты подключать как реальные `<script>`,
а не через `eval` — иначе top-level `const` из `data.js` не попадёт в общую область видимости).
Так проверяются клики, localStorage и синхронизация «Гид» ⇄ «План».

## Ограничения окружения

- **Геолокация** (расстояния в «Гид → Места») и **service worker** (офлайн)
  в браузере работают только на **https или localhost**. По `http://<LAN-IP>` они
  блокируются. Для полноценной работы на телефоне — раздавать по https (напр. хостинг
  вроде Netlify/Cloudflare Pages) или через localhost.
- **Расстояния по дорогам** берутся с публичного демо-сервера OSRM
  (`router.project-osrm.org`) без ключа — годится для пары пользователей; при недоступности
  показывается расстояние по прямой (с «~»).
- **Распознавание по фото** пока не сделано: у DeepSeek API нет vision, нужна отдельная
  vision-модель (Gemini/OpenAI/Claude).

## Внешние сервисы

- **DeepSeek** (`api.deepseek.com`) — чат и переводчик. Ключ хранится в localStorage телефона.
- **open.er-api.com** — курс GEL→RUB.
- **router.project-osrm.org** — дорожные расстояния.
- Навигаторы: Google Maps, Яндекс.Карты, Maps.me (ссылки-маршруты, открываются во внешнем приложении).
