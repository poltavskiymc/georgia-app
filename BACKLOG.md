# Что дальше (backlog)

Приложение живёт на **https://poltavskiymc.github.io/georgia-app/** (GitHub Pages, репо `poltavskiymc/georgia-app`).
Контекст поездки пользователя: **Тбилиси → Батуми → концерт Меладзе → дальше открыто**, едут на машине через границу (Верхний Ларс). Двое пользователей.

## Согласовано сделать (приоритет сверху)

1. **Погода по городам** — прогноз для Тбилиси/Батуми и др. через **open-meteo.com** (бесплатно, без ключа). Полезно на дорогу и open-air концерт. Можно отдельной вкладкой или блоком на «Маршрут»/«Деньги». Не забыть `addTraffic()`.

2. **«Мой план»** — сохранять места из «Маршрута» в свой список, отмечать посещённые/выполненные (чек-лист). Хранение в localStorage. → см. «Синхронизация» ниже.

3. **Гастро-чеклист** — отмечать «попробовано» галочкой. Проще всего встроить в существующую вкладку **«Еда»** (у каждого блюда чекбокс «пробовал», статус в localStorage), а не делать отдельно.

4. **Сборы и граница** — чек-лист для машины и Верхнего Ларса: документы, страховка (грин-карта), аптечка, наличка (лари), техосмотр. Статичный чек-лист с галочками (localStorage).

## Синхронизация плана/чеклиста между 2 устройствами (обсудили, решение НЕ принято)

Идея пользователя: гиперпростой «ключ», который вводят на двух телефонах, и по нему идёт синк; создатель ключа — источник правды, **или планы объединяются**.

**Моя рекомендация:** объединять (merge), а не «источник правды» — оба человека в поездке добавляют места. Модель:
- каждый пункт плана/чеклиста = `{id, text, done, updatedAt}`;
- синк = union по `id`, конфликт разрешается по `updatedAt` (last-write-wins per item);
- «ключ» = короткий код комнаты (room code), по нему читают/пишут общий JSON.

**Бэкенд (нужно выбрать, это outward-facing — спросить пользователя перед деплоем):**
- **Cloudflare Worker + KV** — я могу написать и задеплоить крошечный воркер: `GET/PUT /room/:key` с JSON. Бесплатно, без своего сервера. Синк — поллинг раз в N сек или по кнопке. ← мой выбор.
- **Firebase Realtime DB** — realtime «из коробки», но нужен Firebase-проект (у пользователя есть gmail) и публичный конфиг в клиенте.
- **jsonbin.io / npoint** — совсем просто (bin = ключ), но поллинг и лимиты.
Оффлайн: писать локально, синкать при сети.

## Процесс (важно для следующего чата)

- Правишь css/js → **обязательно** подними `CACHE` в `sw.js` (`georgia-vN`→`vN+1`) и `APP_VERSION` в `js/settings.js`, иначе кэш отдаст старое. Новый файл — ещё и в список `ASSETS` в `sw.js`.
- Проверка без браузера: `node --check js/*.js` + прогон модулей с DOM-заглушкой (`scratchpad/load-test.js`, добавляй новый модуль в его `order`). **Headless Chrome тут зависает — не использовать.**
- Деплой: `git add -A && git commit && git -c credential.helper='!gh auth git-credential' push origin main`. Pages пересобирается ~1 мин. Проверка live: `curl .../js/settings.js | grep сборка`.
- Архитектура и конвенции — в `README.md`.
